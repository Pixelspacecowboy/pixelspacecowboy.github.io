Book Tracker App

Database-driven CLI application demonstrating structured data storage, secure SQL execution, and CRUD operations using Python and SQLite.

Project Overview

This project demonstrates database design and application development through the creation of a Python-based command-line application that manages book records using an SQLite database.

The application allows users to store, retrieve, update, and delete book entries while maintaining a structured and normalized database schema.

Problem the Project Solves

Applications that manage collections of records require reliable storage, retrieval, and modification of structured data. Without proper database design and safe query handling, applications can become difficult to maintain or vulnerable to security issues.

This project demonstrates how a database-backed application can be built with proper schema design, safe query execution, and structured CRUD functionality.

Key Features

  • Normalized SQLite database schema for storing book records
  • Secure SQL queries using parameterized statements
  • Full Create, Read, Update, Delete (CRUD) functionality
  • Input validation for improved reliability
  • Edge case testing including duplicates and null values

Engineering Focus

This project highlights how database design and application logic work together to create reliable software systems. It demonstrates the importance of structured schemas, safe query execution, and data validation when building applications that manage user data.
