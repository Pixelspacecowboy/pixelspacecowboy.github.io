Project Overview

This project showcases my database development skills through the creation of a Python-based command-line application that interacts with an SQLite database. The application was designed to manage book records through structured data storage and user-friendly CRUD operations.

The enhanced version of the project focuses on safer SQL execution, stronger input validation, and improved usability, demonstrating how database-backed applications can be made more secure, reliable, and maintainable.

Problem the Project Solves

Applications that manage structured information need reliable ways to store, retrieve, update, and delete records. Without proper database design and safe query handling, systems can become difficult to maintain and vulnerable to data integrity or security issues.

This project addresses that challenge by implementing a normalized SQLite schema and secure, parameterized database operations that support accurate record management and safer user input handling.

Key Features

  • Designed a normalized SQLite database schema for storing and managing book records
  • Implemented parameterized queries to reduce SQL injection risk
  • Added input validation to improve data quality and application reliability
  • Built Create, Read, Update, and Delete (CRUD) functionality for record management
  • Tested edge cases including null values, invalid inputs, and duplicate entries
